# Evolution — How This Constitution Changes

**Audit ID**: EVO-001

---

## Amendment Process {#amendment-process}

1. Any agent can propose an amendment via PR
2. Amendment must include: what changes, why, impact on existing policies
3. Human must approve via PR review
4. IronCurtain recompiles policies after merge
5. Vectimus reloads policies automatically

## Versioning {#versioning}

- Constitution version = date of last amendment (YYYY-MM-DD)
- Each file has an `Audit ID` at the top
- Breaking changes require a new major version

## Review Cycle {#review-cycle}

- The constitution is reviewed every 30 days
- Review produces: what worked, what didn't, what needs to change
- Review is stored in Engram as a level 2 memory
