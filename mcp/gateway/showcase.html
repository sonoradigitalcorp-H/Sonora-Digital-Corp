⚡ One Gateway to Rule Them All

Instead of nginx → REST API → orchestrator → tools (4 layers), we have one MCP Gateway that handles auth, routing, capabilities, and execution.

curl https://sonoradigitalcorp.com/api/health
{"status":"ok","services":{"postgres":true,"qdrant":true,...}}

curl -X POST https://sonoradigitalcorp.com/api/auth/token \
  -d '{"client_id":"sdc-core","client_secret":"..."}'
# → JWT RS256 token with scope-based auth

curl -X POST https://sonoradigitalcorp.com/api/capability/resolve \
  -H "Authorization: Bearer <token>" \
  -d '{"task":"generate a sales proposal"}'
# → {"capability":"Sales Execution","confidence":0.55}
