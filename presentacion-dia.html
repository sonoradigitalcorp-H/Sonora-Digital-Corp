🔄 El Problema Esta Mañana

¿Por qué estábamos haciendo todo esto?
!

Python 3.10 vs 3.11

🔴 Crítico

El VPS corría Python 3.10, pero el código usaba sintaxis 3.11 (StrEnum, datetime.UTC). Los tests se rompían al borrar caché. Solución temporal: parche con sed en 12 archivos. Solución real: contenedores.

!

Ruff: 911 errores de lint

🔴 Crítico

El código tenía 911 errores detectados por ruff. Solo 627 eran auto-fixables. Los restantes incluían bare-except, raise-without-from, imports sin usar, y variables ambiguas.

!

Systemd + Docker híbrido

🟡 Medio

Jarvis Core y WebUI corrían como servicios systemd nativos, mientras los datos (Neo4j, Redis, Postgres) corrían en Docker. Sin healthchecks consistentes, sin orquestación unificada.

!

Gobernanza en papel

🟡 Medio

Las reglas existían en archivos .md (CONDUCT.md, 10-RULES.md) pero la IA podía saltárselas con --no-verify. Sin branch protection, sin CI gates impositivos.

🏗️ Las 4 Fases de Infraestructura

De la dependencia del sistema operativo a la inmunidad total
Fase 1 · Dockerización

🐳 Python 3.12 en contenedores

Eliminamos la dependencia del Python 3.10 del host VPS. Jarvis Core y WebUI ahora corren en python:3.12-slim. Systemd apagado. 11 contenedores unificados bajo un solo docker-compose.yml.

✅ 432 tests pasan dentro del contenedor en build time
Fase 2 · Redis Streams

⚡ Sistema nervioso en Redis

Reemplazamos la lista Python del orquestador por Redis Streams. El contexto ahora persiste aunque el orquestador se reinicie. Eventos del pipeline se publican dualmente (archivo + stream). Fallback degradado si Redis no está disponible.

✅ 443 tests, 0 failures
Fase 3 · Governance as Code

🔒 La ley es código, no prompt

Branch protection: Tests + Process Gate + Docker Build requeridos. CI fail en errores de lint y baja cobertura. PR template con checklist obligatorio. Auto-assign de reviewers. Push directo a main bloqueado.

✅ Ni el admin puede pushear directo — todo via PR
Fase 4 · Domain Architecture

📊 3 dominios, 11 contenedores

Labels sdc.domain=data|core|ux para todos los contenedores. Script de survey que reporta health por dominio. Redis consumer groups (agents-core, agents-ux) para comunicación entre dominios.

✅ docker compose config válido, build exitoso
