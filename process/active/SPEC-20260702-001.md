## 6. Edge Cases

| EC# | Descripción |
|-----|-------------|
| EC1 | Scraper falla (rate limit, token expirado) → datos stale + alerta |
| EC2 | Token JWT expira → redirect a login sin perder estado |
| EC3 | API caída → frontend muestra fallback data + banner "data may be stale" |
| EC4 | Rate limit Spotify API (429) → retry backoff exponencial |
| EC5 | Dos admins modifican simultáneamente → last-write-wins con timestamp |
| EC6 | Artista sin datos scraper → muestra "Awaiting first sync" |
| EC7 | SSE conexión perdida → reconexión backoff (1s, 2s, 4s, max 30s) |
| EC8 | JSON corrupto → backup automático restaura + notifica admin |

### Stack

**Frontend** (Vercel):
- HTML5 SPA + Tailwind CSS + Framer Motion + Three.js
- SSE nativo (EventSource API) para real-time
- Hash routing (compatible con Vercel)

**Backend** (OVH, systemd):
- FastAPI 0.139+ + uvicorn 0.49+
- sse-starlette para SSE streaming
- aiohttp para scrapers async
- neo4j driver + qdrant-client
- Pydantic v2

## 11. Scale Criteria

- **>10 artistas**: migrar de JSON a PostgreSQL + Qdrant embeddings
- **>50 requests/min**: agregar Redis cache + rate limiting
- **>100k visits/mes**: migrar de SPA vanilla a React/Vite + CDN dedicado
- **>5 usuarios simultáneos en admin**: agregar WebSocket instead of SSE
- **Nuevo artista firmado**: scraper pipeline lo detecta automáticamente
- **Revenue >$1M**: agregar módulo de accounting con QuickBooks integration
