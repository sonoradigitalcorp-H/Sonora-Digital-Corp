# ADR-20260701-003 — Live Data Pipeline via Public Collectors

## Context

ABE Music tiene 3 artistas reales con datos estáticos en `data/abe-music.json` (Hector Rubio: 115M streams, $460K revenue). El pipeline de ventas tiene 0 leads reales porque los datos no fluyen vivos. Necesitábamos datos actualizados sin usar API keys de pago (Spotify, etc.).

## Decision

Crear un pipeline de datos vivos usando **colectores públicos sin API keys**:

- **Deezer API pública** como fuente primaria (sin autenticación requerida)
- **crw** (Rust scraper, MCP nativo) como alternativa cuando esté disponible
- **Playwright MCP** para fuentes JS-heavy (TikTok, YouTube)
- **Fallback Python** (httpx + búsqueda) si crw no responde

Arquitectura: `collector → validar schema → backup → merge → escribir → emitir evento`

## Options Considered

| Opción | Pros | Contras |
|--------|------|---------|
| Spotify API key | Datos oficiales, ricos | $mensual, setup OAuth, riesgo de rate limits |
| Deezer API pública | Sin auth, datos gratis, REST simple | Menos cobertura que Spotify |
| crw (Rust scraper) | MCP nativo, ~50MB RAM, rápido | Compilación Rust, no probado en VPS |
| Python scraping (bs4) | Familiar, fácil de debug | Más lento, frágil a cambios HTML |

Se eligió **Deezer como primario + crw como ideal + Python como fallback** — maximiza disponibilidad sin comprometer simplicidad.

## Consequences

**Positivo:**
- $0 en API keys para datos de artistas
- Datos actualizados cada 6h automáticamente
- Pipeline de revenue puede generar leads basados en métricas reales
- Enterprise Score puede subir con eventos de `data_sync_completed`

**Trade-offs:**
- Deezer puede no tener todos los artistas (cobertura ~80% vs Spotify)
- Datos de Deezer no incluyen streams exactos (solo ranking)
- crw añade complejidad de compilación Rust si no hay binary prebuilt

## Lessons

- La API pública de Deezer es sorprendentemente completa sin auth
- TDD + SPEC + Score + Gherkin = ~30 min de overhead pero estructura sólida
- Los tests con mocks permitieron validar toda la lógica de merge/backup sin llamadas reales
