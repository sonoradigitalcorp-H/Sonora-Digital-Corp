# ADR — Cognitive Kernel: Foundations Architecture

| Campo | Valor |
|-------|-------|
| **ID** | ADR-20260703-A |
| **Fecha** | 2026-07-03 |
| **Estado** | aceptado |

## Contexto

El sistema tiene verdad dispersa (TRUTH.md + fleet.yml + policies sueltas en CONDUCT.md), eventos fragmentados (15 events.jsonl en process/completed/), ejecución sin gates obligatorios, y conocimiento sin conectar (engram.db lineal, Qdrant sin relaciones). Necesitamos una arquitectura que unifique sin romper lo existente.

## Decisión

Adoptar arquitectura de **Cognitive Kernel** con 6 módulos incrementales:

1. **Truth YAML**: Migrar toda la verdad a `truth/` directory con archivos YAML por dominio, schema fijo, y jerarquía de resolución de conflictos. Reemplaza TRUTH.md como fuente de verdad para agentes.

2. **Universal Event Bus**: Unificar todos los eventos en `state/events/events.jsonl` con schema fijo y validación. Reemplaza los 15 events.jsonl dispersos.

3. **Planning Gate**: Script obligatorio que fuerza descomposición, estimación, y validación contra truth/ antes de cualquier ejecución.

4. **Verification Pipeline**: 3 gates (truth, security, cost) que detienen ejecución si algo viola las reglas.

5. **Knowledge Graph**: Poblar Neo4j existente con nodos y relaciones desde eventos + specs + ADRs.

6. **CONDUCT.md + CI**: Actualizar documentación y automatizar validación.

## Por qué YAML y no Markdown

- Markdown es para humanos. YAML es para humanos + máquinas.
- Los agentes pueden parsear YAML sin depender de NLP.
- La validación de schema es trivial (yaml.safe_load + jsonschema).
- Las referencias cruzadas entre archivos son posibles con key:value.

## Por qué event bus unificado y no archivos por spec

- 15 archivos separados no se pueden consultar transversalmente.
- Un solo stream permite replay, migración a base de datos, y análisis temporal.
- La rotación automática evita archivos gigantes.
- El schema fijo permite validación en emisión, no en consumo.

## Consecuencias

- Positivas: Verdad única, eventos consultables, gates evitan errores, grafo conecta todo
- Negativas: Curva de aprendizaje inicial para contribuidores, overhead de gates
- Mitigación: Los gates aplican solo a tier 2+, no a hotfixes

## Alternativas descartadas

| Alternativa | Razón de rechazo |
|-------------|------------------|
| Mantener TRUTH.md + agregar más secciones | Ya tiene 295 líneas, inmanejable, sin schema |
| Base de datos SQL para eventos | Overkill para el volumen actual, events.jsonl es suficiente |
| Kafka/RabbitMQ para event bus | El VPS tiene 11GB RAM, no corre Kafka. events.jsonl + rotación alcanza |
| Kubernetes para agentes | Overkill para 5 agentes en un solo VPS |
