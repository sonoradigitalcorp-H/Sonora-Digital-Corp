## 1. Objetivo

Migrar de verdad dispersa (TRUTH.md + fleet.yml + policies sueltas), eventos fragmentados (15 events.jsonl), y ejecución sin gates, a una arquitectura de Cognitive Kernel con Truth YAML unificada, Event Bus universal, Planning Gate obligatorio, Verification Pipeline, y Knowledge Graph inicial en Neo4j — todo traceable desde Feature hasta Deploy, unificado en OVH VPS, sin duplicar lógica, y sin romper reglas existentes.

---

## 2. Value Driver

**founder-independence**: Elimina dependencia del fundador para saber dónde está la verdad, qué pasó, y si algo viola reglas. **automation**: Gates reemplazan revisión manual. **knowledge**: Grafo conecta todo. **reliability**: Sin eventos perdidos, sin políticas ignoradas, sin ejecución sin plan.

## 6. Edge Cases

- [EC1] truth/ YAML conflictos entre archivos → detector debe reportar error específico
- [EC2] Evento malformado → emit-event.py rechaza con error claro y código
- [EC3] Planning Gate sin agente disponible → plan en estado "pending"
- [EC4] Verification Gate con truth no disponible → skip con warning loggeado
- [EC5] Neo4j caído → populate-graph.py reintenta con backoff (3 intentos, 5s)
- [EC6] >10k eventos en una sesión → rotación automática sin pérdida
- [EC7] Tests existentes (79) se rompen con nueva estructura → migración backward-compatible
- [EC8] VPS sin RAM para Neo4j + queries → limitar población a specs activos
- [EC9] Modelo de agente no disponible → fallback a modelo por defecto desde truth/

## 7. Technical Approach

### Arquitectura

```
truth/ (YAML, git-tracked)
  └── leído por Truth Guardian + Planning Gate + Verification Pipeline

state/events/events.jsonl (append-only, VPS)
  └── escrito por emit-event.py (CLI wrapper con validación)

process/active/PLAN.yaml (generado por plan-gate.py)
  └── verificado por verify-gate.py contra truth/

Neo4j (Docker en VPS, 127.0.0.1:7687)
  └── poblado por populate-graph.py desde events.jsonl + specs
```

### Unificación (sin duplicar)

| Componente | Dónde corre | Trigger |
|------------|-------------|---------|
| Truth YAML | Git (repositorio) | Manual + PR |
| Event Bus | state/events/ en VPS | Cada operación |
| Truth Guardian | systemd en VPS (activo) | Cada 60s |
| Planning Gate | CLI (laptop + VPS) | Antes de ejecutar |
| Verification Pipeline | CLI + GitHub Actions | Pre-commit + PR |
| Knowledge Graph | Neo4j Docker en VPS | populate-graph.py |
| CI | GitHub Actions | git push |
| Backups | systemd timer en VPS | Cada 24h |

### Model Routing (agentes eligen modelo por tarea)

```yaml
# truth/50-coding.yaml
model_routing:
  code_generation: "opencode-go"

  code_review: "claude-sonnet"
  documentation: "opencode-go"
  research: "deepseek-v4"
  security_audit: "claude-opus"
  cost_optimization: true
```
