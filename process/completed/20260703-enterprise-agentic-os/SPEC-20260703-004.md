## 1. Objetivo

Unificar todos los servicios, secretos, configuración, monitoreo, y agentes del ecosistema Sonora Digital Corp bajo un solo OS orquestado por TRUTH.md (autogenerado desde fleet.yml), con cifrado age, nginx gateway, Prometheus+Grafana, LLM gateway, multi-tenant isolation, agent registry, y compliance auditor — elevando el Enterprise Score de 68 a 87.

---

## 2. Value Driver

**founder-independence**: Elimina la dependencia del fundador para tareas operativas (secretos, healthchecks, compliance). **automation**: Monitoreo 24/7 con alertas automáticas. **knowledge**: Documentación autogenerada y verificable. **reliability**: Sin puertos expuestos, todo cifrado, supervisión continua.

## 6. Edge Cases

- [EC1] **age key perdida**: No se puede recuperar secrets — backup de age-key.txt es crítico
- [EC2] **Servicio no responde**: Truth Guardian debe detectar y alertar, no fallar silenciosamente
- [EC3] **Multi-recipient age**: Si laptop y VPS necesitan compartir secrets, usar `-r` múltiple
- [EC4] **Prometheus overload**: En VPS con 11GB RAM, limitar retention a 7d
- [EC5] **Telegram API down**: Truth Guardian debe loggear local y reintentar
- [EC6] **Tenant sin collection**: Crear automáticamente en Qdrant si no existe
- [EC7] **LLM rate limit**: Gateway debe queue y retry con backoff
- [EC8] **Agent duplicado**: Registry debe rechazar names duplicados con error claro

## 7. Technical Approach

### Arquitectura general

```
fleet.yml (SSOT)
  ├── TRUTH.md (autogenerado)
  ├── Nginx configs (autogenerados)
  ├── Prometheus targets (autogenerados)
  └── Grafana datasources (autogenerados)

Truth Guardian (systemd)
  ├── Drift scanner → compara TRUTH.md con ss/docker ps
  ├── Health checker → /api/v1/status
  ├── Compliance auditor → JR-Lite + Score + CONDUCT.md
  └── Telegram alerts
```

## 8. Dependencies

- **M4 → M1**: Secrets cifrados necesarios antes que Truth Guardian lea auth.json
- **M2 → M1**: Nginx proxy necesario antes que status API sea pública
- **M1 → M3**: Status API es input para Prometheus
- **M3 → M5-M8**: Observabilidad es prerequisito para todo lo demás
- **M5**: Qdrant debe tener collections por tenant
- **M6**: LLM gateway necesita config de modelos desde fleet.yml
- **M7**: Agent registry consume health de Truth Guardian
- **M8**: Config unification es el último — todo lo demás debe existir
