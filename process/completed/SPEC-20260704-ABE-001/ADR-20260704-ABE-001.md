# ADR — Cleanup + Migration: De Ollama a opencode

| Campo | Valor |
|-------|-------|
| **ID** | ADR-20260704-ABE-001 |
| **Fecha** | 2026-07-04 |
| **Estado** | aceptado |

## Contexto

El VPS tiene 11GB RAM sin GPU. Correr 5 modelos LLM en Ollama (~11.5GB) compite con 11 contenedores Docker, ABE Service, n8n, y servicios JARVIS. opencode v1.17.13 ofrece modelos gratuitos (`opencode/deepseek-v4-flash-free`, `mimo-v2.5-free`, `north-mini-code-free`) que corren en cloud, no local.

Además, el `opencode.json` tenía 21 agentes, muchos inactivos, con config duplicada y modelos apuntando a Ollama.

## Decisión

1. **Migrar todos los agentes de Ollama a opencode**: Reemplazar 5 modelos LLM locales por modelos cloud gratuitos de opencode. Mantener solo `nomic-embed-text` en Ollama para embeddings (Truth Guardian).

2. **Reducir agentes de 21 a 4**: mystic (primario), hermes (notificaciones), openclaw (gateway), builder (implementación). Los comandos que referenciaban agentes eliminados se redirigen a mystic.

3. **Eliminar directorios duplicados**: `agents/agents/` y `sonora-enterprise-os/skills/vertical/`.

4. **Crear skills SDD**: 8 skills de proceso formales (`sdd-orchestrator`, `sdd-spec`, `sdd-design`, `sdd-apply`, `sdd-verify`, `sdd-archive`, `auto-doc`, `gsd`) en `skills/process/`.

5. **Reorganizar tools/**: Catálogo por dominio (mcp, system, business, abe) sin mover los `.js` originales de `mcp/tools/`.

## Por qué opencode y no mantener Ollama

- Ollama consume RAM local (~11.5GB para 5 modelos) en un VPS con 11GB total y sin GPU
- opencode corre modelos en cloud gratuitos, zero RAM local
- opencode/deepseek-v4-flash-free es más capaz que deepseek-r1:7b local
- nomic-embed-text (274MB) es suficiente para embeddings y corre local

## Por qué reducir a 4 agentes y no mantener 21

- 17 agentes estaban inactivos o duplicaban funcionalidad
- mystic como agente primario + subagentes vía skill cubre todos los casos de uso
- Menos complejidad de configuración y mantenimiento

## Consecuencias

- Positivas: ~11.5GB RAM liberada, agentes responsivos sin latencia local, configuración más simple
- Negativas: Dependencia de conectividad cloud para LLM, opencode v1.x API puede cambiar
- Mitigación: nomic-embed-text local para embeddings offline, fallback a Ollama si cloud cae

## Alternativas descartadas

| Alternativa | Razón de rechazo |
|-------------|------------------|
| Mantener Ollama para todos los modelos | 11.5GB en VPS de 11GB sin GPU — swap mataría el rendimiento |
| Migrar a API de OpenAI/Claude | Costo recurrente vs opencode gratuito |
| Usar solo nomic-embed-text + un modelo pequeño | deepseek-r1:7b-64k aún pesa 4.68GB — opencode cloud es más capaz sin costo de RAM |
