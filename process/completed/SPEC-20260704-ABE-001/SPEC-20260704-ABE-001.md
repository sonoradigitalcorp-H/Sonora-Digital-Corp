## 1. Objetivo

Ordenar la arquitectura de agentes/skills/tools del proyecto y migrar de Ollama a opencode-go + fal.ai, eliminando ~12GB de modelos locales y reduciendo la complejidad operativa como base para ABE Music OS.

---

## 2. Value Driver

Founder-independence + FinOps efficiency + Operational simplicity.

---

## 7. Technical Approach

### Fase 1 — Filesystem Cleanup
```bash
grep -r "agents/agents" apps/ config/ scripts/ --include="*.py" --include="*.json" --include="*.js"
# Si no hay referencias:
rm -rf agents/agents/
rm -rf sonora-enterprise-os/skills/vertical/
```

### Fase 2 — opencode.json Rewrite
Estructura final de agentes: mystic (primary), hermes (subagent), openclaw (subagent), builder (subagent).
17 agentes eliminados se documentan como skills.

### Fase 3 — Skills/Process Creation
Crear `skills/process/` con 8 archivos `.skill.md` que referencian el pipeline SDD.

### Fase 4 — Tools Reorganization
No mover archivos físicamente para no romper imports. Crear estructura de directorios con symlinks.

### Fase 5 — Ollama Cleanup
```bash
ollama rm qwen3:4b-64k qwen2.5:1.5b-32k qwen3:1.7b-32k deepseek-r1:7b-64k llama3.2:3b-64k
```
