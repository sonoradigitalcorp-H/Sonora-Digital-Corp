# Glosario — Sonora Digital Corp × YAMI

| Término | Definición |
|---------|-----------|
| **SDD** | Spec-Driven Development: 11-step cycle (Revenue Gate → Archive) |
| **ADR** | Architecture Decision Record: documento de decisión técnica aprobado |
| **BDD** | Behavior-Driven Development: especificación con Gherkin (Given/When/Then) |
| **TDD** | Test-Driven Development: escribir test antes del código |
| **Revenue Gate** | Validación inicial de que una feature genera valor de negocio |
| **Spec** | Documento que describe qué construir y por qué, con escenarios BDD |
| **Plan** | Desglose del spec en tareas técnicas concretas |
| **OpenRouter** | API unificada de LLMs (Qwen, Claude, DeepSeek, etc.) |
| **OpenCode** | Agente CLI de desarrollo autónomo (reemplaza a Copilot) |
| **MCP** | Model Context Protocol: conexión entre agentes y servicios externos |
| **4R** | Risk, Readability, Reliability, Resilience — quality gates de PR |
| **Memory** | `memory/` — aprendizaje del sistema, NO fuente de verdad |
| **Truth** | `constitution/TRUTH.md` — jerarquía de autoridad del conocimiento |
| **Collab** | Colaborador del repo (Noel + Mystic) |
| **Brain Gateway** | API futura que unifica dominios sin mergear repos |
| **GBrain** | Sistema de agentes previo de YAMI (9 agents registrados) |
| **Engram** | Base de conocimiento vectorial de YAMI (466 entradas) |
| **Enterprise Score** | Métrica compuesta de salud del ecosistema SDC |
| **Lección** | Entrada en `memory/lecciones.json` — aprendizaje post-sprint |
| **Patrón** | Entrada en `memory/patrones.md` — solución recurrente documentada |
