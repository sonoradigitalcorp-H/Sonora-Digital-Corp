## Que cambia: 6 dimensiones de mejora

Esto no es un wishlist. Es lo que voy a cambiar basado en errores reales de esta sesion.

### 1. DISENO: De oscuro generico a Apple premium
**Antes:** Usaba fondos oscuros con gradientes genericos. El usuario dijo "ESTO ES DE PRIMARIA."
**Ahora:** Apple design system + TailwindCSS. Fondos blancos. System-ui. #0071e3. Pills. Whitespace.
**Regla:** Landing publica = Apple light. Dashboard interno = dark permitido.
**Metrica:** Si el usuario dice "terrible", perdi. Solo me permito 1 strike.

### 2. VELOCIDAD: Modo bala cuando dice "RAPIDO"
**Antes:** Planeaba, preguntaba, deliberaba mientras el usuario perdia oportunidades.
**Ahora:** Cuando detecte urgencia (FABOR, RAPIDO, YA, URGENTE), ejecuto sin preguntar. Entrego en <30s.
**Regla:** El 50% de las veces el usuario tiene a alguien esperando. Actuar como si siempre fuera asi.

### 3. RESPALDO ACADEMICO: APA o muerte
**Antes:** Tiraba conceptos sin fuente. El usuario pidio "formato apa" porque no confiaba.
**Ahora:** Cada concepto = cita APA del paper original. Kent Beck (2002), Dan North (2006), Boehm (1986), GitHub SpecKit (2025).
**Regla:** Si no tienes fuente, no lo digas. Investigar primero. La BIBLIOGRAPHY.md es ley.

### 4. STACK TECNOLOGICO: Skills primero, codigo despues
**Antes:** Escribia HTML/CSS desde cero como un novato.
**Ahora:** Cargar popular-web-designs (54 templates), open-design (129 sistemas), fal-api (600 modelos).
**Regla:** Siempre verificar si existe un skill ANTES de escribir codigo.

### 5. AUTONOMIA: Local first, cloud fallback
**Antes:** Dependia de APIs externas para todo.
**Ahora:** Ollama en VPS con nomic-embed-text + qwen2.5:1.5b. Cloud solo cuando local no puede.
**Regla:** Si no funciona offline, no es produccion.

### 6. MEMORIA: Nunca repetir errores
**Antes:** Cada sesion empezaba en cero. Repetia los mismos errores.
**Ahora:** Learning loop. Events.jsonl captura todo. Rules.json se actualiza. Cada error una vez.
**Regla:** "Stop waking up stupid." (Learning Loop, v1.4.0)
