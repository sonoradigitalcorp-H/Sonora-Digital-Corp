# Get Shit Done (GSD)

**Parent OS**: Agent OS
**Tier**: 2
**Description**: Flujo completo de productividad: estado actual → qué hacer → plan → ejecutar → verificar → documentar

## Pipeline
1. **Estado actual** — revisar el proyecto, identificar qué necesita atención
2. **Qué hacer** — priorizar tareas basado en impacto
3. **Plan** — crear plan de acción detallado
4. **Ejecutar** — implementar siguiendo SDD Apply
5. **Verificar** — validar con SDD Verify
6. **Documentar** — archivar con SDD Archive

## Trigger
`/gsd` command en opencode
