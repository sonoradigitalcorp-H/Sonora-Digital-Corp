# DELIVERY OS V1 — Sonora Digital Corp

**Propósito**: Cada cliente recibe valor visible todas las semanas. No hay cliente olvidado.

---

## PRINCIPIO FUNDAMENTAL

**Si el cliente no ve el valor, el valor no existe.**

No importa cuánto código corras en servidores. Si el cliente no recibe un entregable claro, semanal, medible — no estás entregando valor.

## DELIVERY CADENCE

| Frecuencia | Qué | Cómo |
|------------|-----|------|
| **Semanal** | Reporte de entregas | Documento markdown en carpeta del cliente |
| **Mensual** | Reporte de resultados | Reporte con métricas + logros + próximos pasos |
| **Trimestral** | Business Review | Llamada de 30 min: qué funcionó, qué mejorar, próximos objetivos |

## DELIVERY CHECKLIST SEMANAL

Para CADA cliente, cada semana:

- [ ] ¿Qué se entregó esta semana? (listar)
- [ ] ¿El cliente lo recibió y lo vio?
- [ ] ¿Funciona correctamente?
- [ ] ¿Hay feedback del cliente?
- [ ] ¿Hay issues que escalar?
- [ ] ¿El health score cambió?
- [ ] Reporte semanal escrito y enviado

## REGLAS DE DELIVERY

1. **Si no está escrito, no pasó.** Todo entregable se documenta.
2. **Si el cliente no lo vio, no existe.** Envía el reporte. Confirma que lo leyó.
3. **Una semana sin entregable = un paso hacia el churn.**
4. **El delivery es tu mejor venta.** Cliente feliz = referral + upsell.
